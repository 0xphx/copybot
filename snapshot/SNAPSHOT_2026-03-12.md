## find_wallets.py (aktueller Stand)

- Quelle: **GMGN.AI** (nicht mehr Dune Analytics)
- Methode: Browser-Bridge via Claude-Extension (Cloudflare-sicher)
  - Python startet lokalen Server auf Port 9876
  - Claude fuehrt JS-Fetch direkt im GMGN-Tab aus
  - Kein CORS-Problem, kein Cloudflare-Block
- `--replace` ist **deaktiviert** – bestehende Wallets bleiben immer erhalten
- Vorhandene Wallets werden beim Filtern automatisch ausgeschlossen
- Backup wird vor jedem Schreiben als `.json.bak` angelegt

**Aufruf:**
```bash
python find_wallets.py              # Vorschau
python find_wallets.py --apply      # Neue hinzufuegen
python find_wallets.py --period 30d # Anderen Zeitraum nutzen
python find_wallets.py --tag fresh_wallet
```

**Filter-Einstellungen:**
```python
MIN_WIN_RATE      = 0.55   # 55% Win-Rate
MIN_TRADES        = 20     # 20 Trades im Zeitraum
MIN_REALIZED_PNL  = 500    # $500 Realized PnL
MAX_AVG_HOLD_MIN  = 120    # max 2h Haltedauer
MIN_AVG_HOLD_MIN  = 2      # min 2min (kein Ultra-Sniper)
DAYS_ACTIVE       = 3      # aktiv in letzten 3 Tagen
MAX_WALLETS       = 20     # max 20 neue Wallets
```

## Observer-Modus Verhalten

| Auslöser | Preis | Grund |
|---|---|---|
| Wallet verkauft | aktueller Marktpreis | `WALLET_SOLD` |
| Kein Preischange > 30 Min | aktueller Marktpreis | `OBSERVER_STAGNATION` |
| Position > 60 Min offen | aktueller Marktpreis | `OBSERVER_MAX_HOLD` |
| 10x kein Preis / 30 Min kein Preis | 0 EUR (Totalverlust) | `PRICE_UNAVAILABLE` |
| Manueller Shutdown (Ctrl+C) | aktueller Marktpreis | `SESSION_ENDED` |
| Reboot/Crash (naechster Start) | 0 EUR (price_missing) | `CRASH_RECOVERY` |

`SESSION_ENDED` und `CRASH_RECOVERY` werden gespeichert aber aus ALLEN Stats-Berechnungen herausgefiltert.

## Confidence Score Formel

```
wr_score    = win_rate * 0.45
count_score = min(trades / 50, 1.0) * 0.20
pnl_score   = log2(1 + max(avg_pct_capped, 0) / 25) / log2(21) * 0.35  (Winsorized 95%)
confidence  = clamp(wr_score + count_score + pnl_score, 0.0, 1.0)
Minimum 5 Trades -> sonst 0.2
SESSION_ENDED + CRASH_RECOVERY herausgefiltert
```

## SL/TP Prioritätskette (Analysis-Modus)

```
1. Dynamische Werte aus Analysis-DB (ab 20 sauberen Trades)
2. Aus Observer-DB berechnet (wenn >= 15 saubere Trades)
   -> TP: 25. Perzentil der Gewinne (Winsorized 95%)
   -> SL: 75. Perzentil der Trade-Lows
3. Label-basierte Defaults (STRATEGY_SL_TP_DEFAULTS)
4. Globale Defaults (-50% / +100%)
```
