## Observer-Modus Timeouts (aktueller Stand)

| Timeout | Wert | Anpassbar? |
|---|---|---|
| Stagnation (kein Preischange) | **15 Min** (war 30) | Nein (Konstante) |
| Max-Haltedauer | **60 Min Default** | **Ja, beim Start konfigurierbar** |
| Anti-Softlock: keine Preisfehler | 10x | Nein |
| Anti-Softlock: kein Preis | 30 Min | Nein |

**Konfiguration beim Start (Observer Mode):**
```
 Max. gleichzeitige Positionen [5]:
 Max-Haltedauer pro Position (Minuten) [60]:   ← NEU
  Connection Timeout (Sekunden) [30]:
```

## Rate-Limiting Fix (solana_polling.py)

Polling läuft jetzt in dynamischen Batches statt alle Wallets gleichzeitig:

```python
batch_size = max(3, min(15, len(wallets) // 4 or 3))  # ~4 Batches pro Durchlauf
RPC_MAX_RPS = 8                                          # konservativ für kostenlosen RPC
min_delay   = batch_size / RPC_MAX_RPS                  # Pause zwischen Batches
batch_delay = max(batch_delay, min_delay)
```

**Mit 40 Wallets:** 4 Batches × 10 @ 1.25s Delay → ~8 req/s → kein Rate-Limit mehr.
